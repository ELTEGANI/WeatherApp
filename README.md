## Overview

Welcome! This project is part of the interview process for the Android developer role. You are required to complete a Android (Kotlin) assessment, following the guidelines and constraints below. Please read this document carefully before starting.

## Assessment Instructions

- **Complete the assessment in Android using Kotlin and JetPack Compose.**
- **Do not use ChatGPT, Claude or any AI tool for code generation. We encourage the use of AI but this is an assessment aiming at evaluating your very own knowledge, and crafting skills.**
- **Do not use DreamFlow or AI for UI generation.**
- **Third-party dependencies** (from MavenCentral, Jit, etc) are allowed only for:
  - Cross-cutting concerns (logging, linting, dependency injection)
  - Platform resource access (location, contacts, etc.)
  - Serialization/Deserialization
  - _Not_ for functional or UI components.
- **Prefer custom UI Widgets**
- **Prefer Kotlin DSL over Groovy**
- **Architecture and state management options:** MVVM, MVC, MVI, Clean architecture, StateFlow
- **Provide a comprehensive README** (this file) including:
  - Conventions, architecture, and general considerations
  - Exhaustive list of third-party packages and their purposes
  - Build instructions
  - Additional notes to demonstrate your knowledge


## Submission

- **Timeline:** 3 days to complete the assessment (or sooner if possible).
- **Repository:** Please upload the complete solution to the Git repository (GitHub, Bitbucket, or GitLab) that you were invited to for this assessment.
- **Content:** Ensure your repository contains all source code, assets, and a README.
- **Branch:** Ensure that your source code is committed to the main branch of the repository.

If you have any questions, please reach out to the assessment coordinator.
